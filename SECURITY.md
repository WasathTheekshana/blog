## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **Do NOT** open a public GitHub issue for security vulnerabilities
2. Email the details to: **wasath.vt@gmail.com**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Resolution Timeline**: Depends on severity

### Severity Levels

| Level | Description | Response Time |
|-------|-------------|---------------|
| Critical | Immediate threat, data exposure | 24-48 hours |
| High | Significant risk | 1 week |
| Medium | Limited impact | 2 weeks |
| Low | Minimal risk | 1 month |

## Security Best Practices

When contributing to this project:

- Never commit secrets, API keys, or credentials
- Keep dependencies updated
- Follow secure coding practices
- Report any suspicious activity

## Hall of Fame

We appreciate security researchers who help keep this project safe. Contributors who report valid vulnerabilities will be acknowledged here (with permission).
